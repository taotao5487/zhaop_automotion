# 招聘抓取与公众号草稿箱操作指南

这份文档只讲一件事:

- 运行时先点哪里
- 终端里输入什么命令
- 每一步跑完后看什么结果

适用目录:

```bash
cd /Users/xiongtao/Documents/zhaop_automotion
```

## 一、先做一次环境准备

第一次用时先执行:

```bash
cd /Users/xiongtao/Documents/zhaop_automotion
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp env.example .env
```

如果你已经配好环境，后面每次运行前通常只要:

```bash
cd /Users/xiongtao/Documents/zhaop_automotion
source .venv/bin/activate
```

## 二、微信服务先启动

爬虫项目里“推送到公众号草稿箱”和微信项目里的 RSS 拉文，都依赖微信服务先启动。

### 1. 启动服务

终端输入:

```bash
cd /Users/xiongtao/Documents/zhaop_automotion
source .venv/bin/activate
python app.py
```

### 2. 需要点击的页面

服务启动后，浏览器打开:

- 登录页: `http://localhost:5001/login.html`
- 订阅管理页: `http://localhost:5001/rss.html`
- 接口文档页: `http://localhost:5001/api/docs`

### 3. 先确认服务正常

新开一个终端，输入:

```bash
cd /Users/xiongtao/Documents/zhaop_automotion
source .venv/bin/activate
curl "http://localhost:5001/api/health"
curl "http://localhost:5001/api/admin/status"
```

看到健康状态正常、登录状态正常，再继续下面操作。

## 三、爬虫项目操作指南

## 1. 测试单医院

### 最推荐

先测试配置，不入库:

```bash
python main.py --test "重庆市人民医院" -v
```

如果医院名不确定，也可以直接用列表页 URL:

```bash
python main.py --test "https://example.com/recruit/list" -v
```

### 真正跑单医院

```bash
python main.py --site "重庆市人民医院"
```

### 如果想先诊断列表页选择器

```bash
python scripts/test_hospital_url_interactive.py
```

然后把医院招聘列表页 URL 粘贴进去。

## 2. 批量爬取

### 先小范围试跑

```bash
python main.py --all --site-limit 3
```

### 全量批量爬取

```bash
python main.py --all
```

### 查看当前数据库统计

```bash
python main.py --stats
```

## 3. 批量爬取并进入审核 URL

这是“批量爬取 + 打开审核页面 + 审核后继续详情增强”的命令。

### 跑全部医院并进入审核页

```bash
python main.py --all --with-details --select-detail-urls
```

### 联调时只处理前 3 条详情

```bash
python main.py --all --with-details --details-max-items 3 --select-detail-urls
```

### 运行时你要点击什么

1. 命令运行后，程序会自动启动本地审核页面。
2. 在页面里点开原文，判断这条是不是你要保留的招聘。
3. 对每条候选做保留或丢弃。
4. 最后点击页面底部的“完成并继续”。

点完“完成并继续”后，终端会自动继续跑，不需要重新执行 `main.py`。

## 4. 附件的链接转换

这一步的意思是:

- 把待审包里的附件上传到附件服务
- 回填公众号文章里可用的附件链接

### 单篇待审包处理

```bash
python scripts/upload_wechat_attachments.py --review-dir "/absolute/path/to/data/exports/wechat_review/<时间戳>/<文章目录>"
```

默认附件服务是 `idocx`。如果你要指定:

```bash
python scripts/upload_wechat_attachments.py --review-dir "/absolute/path/to/data/exports/wechat_review/<时间戳>/<文章目录>" --service idocx
```

### 跑完后会生成

- `attachment_links.json`
- `attachment_links.md`
- 更新后的 `article.html`
- 更新后的 `package.json`

## 5. 附件的岗位表转为图片以及成稿

这一步不是单独再点一个新脚本，通常是在详情增强时自动完成“生成待审包和成稿”。

### 推荐命令

```bash
python main.py --site "重庆市人民医院" --with-details --select-detail-urls
```

或者批量:

```bash
python main.py --all --with-details --select-detail-urls
```

### 你要重点看两个产物目录

- `data/exports/details/<时间戳>/summary.json`
- `data/exports/wechat_review/<时间戳>/<文章目录>/`

其中:

- `summary.json` 适合整批后续推送
- `wechat_review/...` 就是单篇待审包目录 `review_dir`

## 6. 推送到公众号草稿箱

### 推送单篇待审包

```bash
python push_wechat_review_to_official_draft.py --review-dir "/absolute/path/to/data/exports/wechat_review/<时间戳>/<文章目录>"
```

### 按整批 summary 批量推送

```bash
python push_wechat_review_to_official_draft.py --summary-path "/absolute/path/to/data/exports/details/<时间戳>/summary.json"
```

### 如果允许重复推送

```bash
python push_wechat_review_to_official_draft.py --review-dir "/absolute/path/to/data/exports/wechat_review/<时间戳>/<文章目录>" --force
```

## 7. 一键启动，直到完成推送公众号草稿

这里要特别说明:

- 当前仓库没有“从爬虫开始到公众号草稿全部自动结束”的单条命令
- 因为中间有“审核 URL”这一步，需要你人工点击确认
- 当前最接近的一键全流程，是下面这两步

### 第一步: 跑爬虫到生成待审包

```bash
python main.py --all --with-details --select-detail-urls
```

运行后去浏览器审核，最后点击“完成并继续”。

### 第二步: 审核完成后，一键处理最新一轮待审包并批量推送

```bash
./.venv/bin/python scripts/publish_latest_wechat_batch.py --service idocx
```

这个脚本会自动:

1. 找最新一轮 `summary.json`
2. 识别每个 `review_dir`
3. 有附件时先上传并回填
4. 再批量推送到共享公众号草稿串

补充:

- 不建议把昨天的 `summary.json` 时间戳手动写死到命令里
- 如果输出 `warning | summary_path_is_not_latest`，说明你指定的不是最新一轮
- 如果输出 `skipped_duplicate`，说明这批文章之前已经推送过，不会再次写入草稿箱

## 四、微信项目操作指南

微信项目这部分的核心是:

1. 拉取招聘文章
2. 确认哪些是未推送文章
3. 推送文章到公众号草稿箱
4. 尽量用最少命令跑完整条链路

## 1. 拉取招聘文章

### 先点击登录页扫码

浏览器打开:

```text
http://localhost:5001/login.html
```

### 如果还没订阅公众号

先搜索公众号:

```bash
curl "http://localhost:5001/api/public/searchbiz?query=人民日报"
```

再把返回结果里的 `fakeid` 拿来订阅:

```bash
curl -X POST "http://localhost:5001/api/rss/subscribe" \
  -H "Content-Type: application/json" \
  -d '{"fakeid":"MzA1MjM1ODk2MA==","nickname":"人民日报"}'
```

### 手动拉一次最新文章

```bash
curl -X POST "http://localhost:5001/api/rss/poll"
```

## 2. 确认未推送文章

这一步就是查已经筛成 `confirmed`、但还没推送进共享草稿串的文章:

```bash
curl "http://localhost:5001/api/recruitment/export?format=json&status=confirmed&push_status=unpushed&profile=title_url"
```

如果你想看更完整内容:

```bash
curl "http://localhost:5001/api/recruitment/export?format=json&status=confirmed&push_status=unpushed&profile=full"
```

## 3. 推送文章

### 推送最新一篇未推送文章

```bash
curl -X POST "http://localhost:5001/api/recruitment/official-draft/series/push"
```

### 推送指定原文链接对应的文章

```bash
curl -X POST "http://localhost:5001/api/recruitment/official-draft/series/push?source_url=https://example.com/recruit/123"
```

### 查看当前共享草稿串状态

```bash
curl "http://localhost:5001/api/recruitment/official-draft/series/status"
```

## 4. 一键启动到推送到公众号草稿的全部运行

这里也要说明一下:

- 当前仓库没有“微信 RSS 拉文到公众号草稿箱”的单条一键脚本
- 目前最省事的做法，就是按下面这 5 步顺序执行
- 真正经常复制的只有第 3 到第 5 步

### 第一步: 启动服务

```bash
python app.py
```

### 第二步: 浏览器扫码登录

打开:

```text
http://localhost:5001/login.html
```

### 第三步: 拉文章

```bash
curl -X POST "http://localhost:5001/api/rss/poll"
```

### 第四步: 查未推送文章

```bash
curl "http://localhost:5001/api/recruitment/export?format=json&status=confirmed&push_status=unpushed&profile=title_url"
```

### 第五步: 推送最新一篇未推送文章

```bash
curl -X POST "http://localhost:5001/api/recruitment/official-draft/series/push"
```

### 最短复制版

```bash
curl -X POST "http://localhost:5001/api/rss/poll"
curl "http://localhost:5001/api/recruitment/export?format=json&status=confirmed&push_status=unpushed&profile=title_url"
curl -X POST "http://localhost:5001/api/recruitment/official-draft/series/push"
curl "http://localhost:5001/api/recruitment/official-draft/series/status"
```
